Headline: four configurations

Unsloth ships dynamic GGUF quants of Qwen3.6's official MTP-enabled checkpoints. The custom llama.cpp build wires the qwen3_next_mtp draft head into its speculative pipeline via the new --spec-type mtp flag. We benchmarked Q4_K_XL (Unsloth-Dynamic, more bits on hot tensors) and Q8_0 (standard, near-lossless) on both the dense 27B and the MoE 35B-A3B.

What the data says

Observations:
  • llama.cpp's MTP genuinely activates the draft head on the GGUF. The startup log confirms loading MTP head from '…UD-Q4_K_XL.gguf' (override_arch=qwen35_mtp) and adding speculative implementation 'mtp'. This is not dead capacity — the speculative pipeline is wired up.
  • MTP gives the biggest lift on the dense 27B at Q8. Standard Q8 GGUF on this hardware (from the Carnice-V2 benchmark) ran 47.6 tok/s. The MTP build at Q8 runs 66.1 tok/s — a +39 % gain from speculation alone. Q4 also lifts (+~20 % vs standard Q4_K_M GGUF).
  • llama.cpp MTP succeeds on the MoE where vLLM MTP didn't. vLLM's qwen3_next_mtp N=1 gave 0 % gain on the 35B-A3B (verify pass already cheap on the MoE). llama.cpp's implementation reaches 178.8 tok/s at Q4 — comparable to vLLM FP8 baselines at 203 — meaning the speculation is paying off even on the MoE.
  • 27B llama.cpp + MTP doesn't catch vLLM's INT4 stacks. 85.7 tok/s on Q4_K_XL is real progress (over 70 tok/s plain Q4 in earlier Carnice measurements), but vLLM's INT4 + DFlash champion at 146 steady is in a different class. The INT4 Marlin kernel on Blackwell plus DFlash's block drafter outpaces llama.cpp's single-draft-token MTP integration.
  • Q4_K_XL > Q8_0 on both models, by ~30 % on 27B and ~7 % on 35B. Standard bandwidth math — less to stream per token. The Unsloth-Dynamic variant keeps quality close to Q5_K for most layers.

Building the llama.cpp MTP image

The MTP support is in PR #22673 (mainline merge pending). We built from am17an/llama.cpp:mtp-clean via the in-repo Dockerfile, exposing --spec-type mtp and --spec-draft-n-max. 

# Build the image (once per host)
docker build -t llama.cpp-mtp:latest \
  -f models/qwen3.6/Dockerfile.llama-mtp models/qwen3.6/

# Run the champion (35B Q4_K_XL + MTP) on port 11435
docker compose -f models/qwen3.6/docker-compose.llama-35b-q4-mtp-rtx.yml up -d

# Or 27B Q4_K_XL + MTP on port 11436
docker compose -f models/qwen3.6/docker-compose.llama-27b-q4-mtp-rtx.yml up -d

Image: nvidia/cuda:13.0.0-runtime-ubuntu24.04 base, ~4.45 GB. Built with -DGGML_CUDA=ON -DCMAKE_CUDA_ARCHITECTURES=120 for SM_120 (Blackwell) and OpenSSL for HF downloads. PR limitations: -np 1 only (no parallel slots with MTP), --mmproj not supported (text-only).
